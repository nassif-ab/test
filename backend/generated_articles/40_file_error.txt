  "contenu": "# Déploiement d'un réseau LTE local avec Open5GS\n\nCe guide explique comment déployer un réseau LTE local (aussi appelé réseau privé LTE) en utilisant Open5GS, une implémentation open-source du cœur de réseau 4G/5G. Un réseau LTE local offre une connectivité haute performance, sécurisée et dédiée pour des applications industrielles, de sécurité publique, ou d'événements. Il est idéal lorsque la couverture cellulaire publique est insuffisante ou lorsque des exigences de latence et de bande passante spécifiques doivent être satisfaites.\n\n## Prérequis\n\n*   **Serveur Linux :** Un serveur Linux avec une adresse IP statique. Ubuntu Server 20.04 LTS est une bonne option.  Il doit avoir au moins 4 Go de RAM et un processeur multi-cœur.\n*   **Carte SIM compatible LTE :** Une carte SIM compatible LTE et programmée avec les informations de votre réseau (PLMN, IMSI, Ki, OPC/OPc). Vous pouvez les obtenir auprès de fournisseurs de cartes SIM LTE ou en créer vous-même, mais cela nécessite une expertise considérable.\n*   **eNodeB (Base Station) :**  Un eNodeB (evolved NodeB), aussi appelé station de base LTE, est nécessaire pour fournir la couverture radio. Les eNodeB peuvent être physiques ou logiciels (e.g., srsRAN).  Assurez-vous de choisir un eNodeB compatible avec les bandes de fréquences que vous souhaitez utiliser et conforme aux réglementations locales.\n*   **Adresse IP Publique (Optionnelle) :** Si vous souhaitez que votre réseau local ait accès à Internet, vous aurez besoin d'une adresse IP publique.\n*   **Connaissances de base des réseaux LTE et des systèmes Linux.**\n\n## Étapes d'installation et de configuration\n\n### 1. Installation d'Open5GS\n\nIl existe plusieurs façons d'installer Open5GS.  L'utilisation du dépôt APT (si disponible pour votre distribution) est la méthode la plus simple.\n\n```bash\nsudo apt update\nsudo apt install open5gs\n```\n\nSi les paquets ne sont pas disponibles via APT, vous pouvez compiler Open5GS à partir des sources, ce qui nécessite l'installation de dépendances supplémentaires. Consultez la documentation d'Open5GS pour des instructions détaillées : [https://open5gs.org/](https://open5gs.org/)\n\n### 2. Configuration d'Open5GS\n\nLes fichiers de configuration d'Open5GS se trouvent généralement dans `/etc/open5gs/`.  Les fichiers de configuration importants incluent :\n\n*   `open5gs-amfd.yaml` : Configuration de l'Access and Mobility Management Function (AMF).\n*   `open5gs-smfd.yaml` : Configuration de la Session Management Function (SMF).\n*   `open5gs-upfd.yaml` : Configuration de l'User Plane Function (UPF).\n*   `open5gs-pcfd.yaml` : Configuration de la Policy Control Function (PCF).\n*   `open5gs-hssd.yaml` : Configuration du Home Subscriber Server (HSS).\n\nModifiez ces fichiers pour correspondre à votre configuration réseau.  Les paramètres importants incluent:\n\n*   **PLMN (Public Land Mobile Network):**  L'identifiant du réseau mobile (MCC + MNC).  Par exemple, `310260` pour un réseau fictif.\n*   **TAC (Tracking Area Code):** Un identifiant unique pour une zone de suivi.\n*   **DNN (Data Network Name):** Le nom du réseau de données (e.g., `internet`, `ims`).\n*   **Adresse IP des interfaces réseaux :**  Assurez-vous que les adresses IP configurées correspondent à la configuration de votre serveur.\n\nExemple de configuration HSS (partielle):\n\n```yaml\nfreeDiameter:\n  ConnectPeer:\n    - addr: 127.0.0.1\n      port: 3868\n\nhss:\n  plmn_list:\n    - mcc: \"310\"\n      mnc: \"260\"\n      mnc_length: 2\n  knssai_list:\n    - sst: 1\n      sd: 1\n\n  ue_pool:\n    - imsi: \"310260000000001-310260000000999\"\n      opc: \"8E28B65192B49D6D6E647E35E85453A1\"\n      ki: \"4658087F0F8B5F6A8D529999F3E6A887\"\n```\n\n### 3. Configuration de l'eNodeB\n\nLa configuration de l'eNodeB dépend du modèle spécifique que vous utilisez.  En général, vous devrez configurer les paramètres suivants :\n\n*   **Fréquence (EARFCN):**  Le numéro de canal absolu de la radiofréquence de l'UTRA Evolved (EARFCN) que l'eNodeB utilise.\n*   **PLMN (Public Land Mobile Network):**  Identique à celui configuré dans le cœur de réseau Open5GS.\n*   **TAC (Tracking Area Code):**  Identique à celui configuré dans le cœur de réseau Open5GS.\n*   **Adresse IP du serveur Open5GS (AMF):**  L'adresse IP du serveur où l'AMF est en cours d'exécution.\n\nPar exemple, si vous utilisez srsRAN, vous devrez modifier le fichier `enb.conf` avec les paramètres appropriés.\n\n### 4. Ajout d'abonnés (Utilisateurs) dans HSS\n\nVous devez ajouter les informations des abonnés (IMSI, Ki, OPC) dans le HSS pour que les UE (User Equipments, c'est-à-dire, les téléphones portables) puissent s'authentifier sur le réseau. Vous pouvez utiliser l'interface Web d'Open5GS (si elle est installée) ou modifier directement la base de données HSS.\n\nPar exemple, avec `open5gs-dbctl` :\n\n```bash\nopen5gs-dbctl subscriber add 310260000000001 --imsi 310260000000001 --opc 8E28B65192B49D6D6E647E35E85453A1 --ki 4658087F0F8B5F6A8D529999F3E6A887\n```\n\n### 5. Démarrage des Services Open5GS\n\nUne fois la configuration terminée, démarrez les services Open5GS :\n\n```bash\nsudo systemctl start open5gs-amfd\nsudo systemctl start open5gs-smfd\nsudo systemctl start open5gs-upfd\nsudo systemctl start open5gs-pcfd\nsudo systemctl start open5gs-hssd\n```\n\nVérifiez l'état des services :\n\n```bash\nsudo systemctl status open5gs-amfd\nsudo systemctl status open5gs-smfd\n# ... et ainsi de suite pour chaque service\n```\n\n### 6. Test de la connexion\n\nAllumez un UE compatible LTE avec la carte SIM configurée et vérifiez qu'il s'enregistre sur le réseau LTE local.  Vous pouvez utiliser l'outil `tcpdump` ou `wireshark` sur le serveur Open5GS pour surveiller le trafic réseau et vérifier l'authentification et l'établissement de session.\n\n### 7. Configuration du routage (si nécessaire)\n\nSi vous avez besoin d'accéder à Internet depuis votre réseau LTE local, vous devrez configurer le routage et le NAT (Network Address Translation) sur le serveur UPF.  Cela implique généralement de configurer des règles `iptables` pour acheminer le trafic vers l'interface réseau connectée à Internet.\n\n## Dépannage\n\n*   **Vérifiez les logs :** Les fichiers journaux d'Open5GS se trouvent généralement dans `/var/log/open5gs/`.  Consultez ces fichiers pour identifier les erreurs.\n*   **Utilisez tcpdump/wireshark :**  Capturez le trafic réseau pour analyser les problèmes de communication entre l'UE, l'eNodeB et le cœur de réseau.\n*   **Vérifiez la configuration de l'eNodeB :** Assurez-vous que l'eNodeB est correctement configuré et qu'il communique avec l'AMF.\n*   **Problèmes de SIM :** Vérifiez que la SIM est correctement programmée et active.  Assurez-vous que l'IMSI, Ki et OPC sont corrects.\n\n## Conclusion\n\nLe déploiement d'un réseau LTE local avec Open5GS peut être complexe, mais il offre une solution de connectivité puissante et flexible pour diverses applications. En suivant ce guide et en consultant la documentation d'Open5GS, vous pouvez configurer votre propre réseau LTE privé et profiter de ses avantages.\n\n## Ressources supplémentaires\n\n*   **Site web Open5GS:** [https://open5gs.org/](https://open5gs.org/)\n*   **Documentation Open5GS:** [https://open5gs.org/open5gs/](https://open5gs.org/open5gs/)\n*   **Projet srsRAN:** [https://www.srslte.org/](https://www.srslte.org/) (pour eNodeB logiciel)\n\nN'hésitez pas à me poser des questions si vous avez besoin d'aide supplémentaire.
